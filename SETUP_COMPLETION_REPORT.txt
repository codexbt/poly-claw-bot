5. SIGNAL SCORING SYSTEM
═══════════════════════════════════════════════════════════════════════════

Three Components (weighted):
  
  1. MOMENTUM (40% weight):
     • Calculates % change over 45-second window
     • Checks tick consistency (prices moving in signal direction)
     • Score: 0.0-1.0 based on magnitude & consistency
  
  2. CANDLE ANALYSIS (35% weight):
     • Detects patterns: engulfing, doji, hammer, shooting star
     • Analyzes body ratio vs total range
     • Counts consecutive same-direction candles
     • Score: 0.0-1.0 based on pattern strength
  
  3. MARKET CONFIRMATION (25% weight):
     • Checks if YES/NO token price >= $0.86
     • Calculates how far above 0.86 (0.86-1.00 range)
     • HARD GATE: if < 0.86, trade is BLOCKED regardless of other scores
     • Score: 0.5-1.0 if market confirms, 0.0 otherwise

FINAL SCORE = (Mom * 0.40) + (Candle * 0.35) + (Market * 0.25)
             = 0.0 to 1.0

TRADE DECISION:
  • Score 0.0-0.50 ........... NO TRADE (insufficient confidence)
  • Score 0.50-0.70 ......... EXECUTE $1-2 trade
  • Score 0.70-1.00 ......... EXECUTE $3 trade

6. ORDER EXECUTION FLOW
═══════════════════════════════════════════════════════════════════════════

When Signal Score > Threshold:

  1. Determine Token:
     • UP signal → BUY YES token (pays $1 if price goes up)
     • DOWN signal → BUY NO token (pays $1 if price goes down)

  2. Set Amount:
     • Based on total score: $1, $2, or $3

  3. Create Market Order:
     • Type: FOK (Fill or Kill)
     • Side: BUY
     • Token ID: YES or NO token from market

  4. Submit Order:
     • Via Polymarket CLOB API
     • Using signed order with private key
     • Returns orderID on success

  5. Track Position:
     • Store entry price
     • Store signal direction (UP/DOWN)
     • Start monitoring for reversal

7. REVERSAL EXIT & POSITION MANAGEMENT
═══════════════════════════════════════════════════════════════════════════

Every Trading Cycle:
  1. For each crypto with open position:
  2. Calculate % change from entry price
  3. Check if reversal occurred:
     • UP signal + price down 0.50% → EXIT
     • DOWN signal + price up 0.50% → EXIT
  4. If reversed: clear entry_price & open_position
  5. Otherwise: log current position status

This provides automatic loss management without selling!

9. ERROR HANDLING
═══════════════════════════════════════════════════════════════════════════

Graceful Failure Points:

Price Fetching Fails:
  → Logs warning, continues with next cycle (no spam of failing cryptos)

Market Not Found:
  → Logs debug info, tries next crypto, retries later

API Timeout:
  → Requests module timeout (5-10s), logs and continues

Order Submission Error:
  → Logs error, does NOT track position, retries next cycle

Daily Limit Reached:
  → Stops trading for rest of day
  → Resets counters at midnight UTC

Keyboard Interrupt (Ctrl+C):
  → Catches gracefully
  → Logs "Bot stopped by user"
  → Exits cleanly
